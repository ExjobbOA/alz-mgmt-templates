## Syfte

Visa att en uppdatering av ALZ-biblioteket absorberas av plattformen utan manuella
tillståndsedit. Kapitel 5 står för det akademiska resonemanget kring K11; detta protokoll
fångar operatörsledet och evidensen.

**Bump:** `2025.09.2` + lokal EncryptTransit-patch → `2026.04.0`
**actionOnUnmanage:** `DetachAll` → `DeleteAll` på governance-stackarna (logging och
networking behålls på `DetachAll` p.g.a. data-bärande resurser).

---

## Context

Baselinen är ren (11/11 succeeded) men bär en känd avvikelse: `Enforce-EncryptTransit`-
varianterna togs bort lokalt ur `lib/alz/` för att kringgå en case-sensitivity-bugg som
blockerade deploys i `2025.09.2`. Enligt Microsofts PR #308-kommentar var endast den
oversionerade varianten faktiskt trasig; de versionspinnade varianterna (`_20240509`,
`_20241211`) fungerade alltid korrekt eftersom de pinnar policy definition-versioner.
Operatörens lokala patch (v1.0.2, SHA `3144344`) tog bort alla tre — en överreaktion.

Uppströmsreaktionen i `2026.04.0` är att ta bort den oversionerade varianten helt
(tillsammans med en bredare deprecation cleanup) och behålla de versionerade. Bumpen +
regenerering korrigerar därmed automatiskt operatörens överreaktion.

Denna körning absorberar två uppströmsförändringar i en rörelse:

1. Nya/ändrade definitioner och assignments i `2026.04.0`
2. Borttagning av deprekerade policies (som med `DeleteAll` nu faktiskt raderas, inte
   orphan-detachas)

### 2.4.5 CD-körning: blockerade deletes på int-root-stacken

Initial CD-körning failade på governance-int-root med:

```
Error: DeploymentStackUpdateFailed
Error: DeploymentStackDeleteResourcesFailed
```

Tre blockerande referenser rapporterades av deployment stack-API:t under cleanup-fasen:

**Fel 1:**
```
The policy definition 'Deny-Storage-minTLS' cannot be deleted. It is referenced
by the policy set definition 'Enforce-EncryptTransit' using version '1.0.0'.
```

**Fel 2:**
```
The policy set definition 'Enforce-Guardrails-KeyVault' cannot be deleted. It is
referenced by the policy assignment 'Enforce-GR-KeyVault' on landingzones MG.
```

**Fel 3 (rapporterades vid re-run efter att fel 2 åtgärdats):**
```
The policy set definition 'Enforce-Guardrails-KeyVault' cannot be deleted. It is
referenced by the policy assignment 'Enforce-GR-KeyVault' on platform MG.
```

Båda blockare härstammar från orphan-resurser som lämnades kvar i Azure efter
operatörens lokala patch i tag v1.0.2 (där `Enforce-EncryptTransit` och
`Enforce-Guardrails-KeyVault` togs bort ur Bicep-templaten). Eftersom stackarna kördes
med `actionOnUnmanage: DetachAll` vid den tidpunkten raderades inte resurserna i Azure
utan blev `DetachedResources` respektive lämnade kvar aktiva assignments på två MG:er.

**Åtgärd — manuell intervention:**

```powershell
Remove-AzPolicySetDefinition `
  -Name "Enforce-EncryptTransit" `
  -ManagementGroupName "alz" -Force

Remove-AzPolicyAssignment `
  -Name "Enforce-GR-KeyVault" `
  -Scope "/providers/Microsoft.Management/managementGroups/landingzones"

Remove-AzPolicyAssignment `
  -Name "Enforce-GR-KeyVault" `
  -Scope "/providers/Microsoft.Management/managementGroups/platform"
```

Efter att samtliga tre orphan-referenser åtgärdats lyckades alla 11 stackar.

### 3.2 Observationer

**Observation 1: What-if täcker inte deployment stack cleanup-fasen.** Trots att git
diff visade 20 borttagna filer i `lib/` rapporterade what-if 0 resource-level deletes
över alla 11 stackar. Den faktiska delete-aktiviteten exponerades först i CD-loggens
cleanup-fas, där tre orphan-resurser blockerade.

**Observation 2: Fork-skuld från v1.0.2 exponerades vid switch till DeleteAll.** Den
lokala patchen i v1.0.2 lämnade orphan-resurser (en orphan policy set + två orphan
assignments på olika MG:er) som DetachAll inte städade upp. Switch till DeleteAll
tvingade fram städningen, men de tre referenserna krävde manuell intervention för att
lossa.

**Observation 3: Versionerade EncryptTransit-varianter listades som modify, inte
create.** Eftersom de fanns kvar som DetachedResources i baseline (resultat av tidigare
DetachAll-körningar) adopterades de tillbaka som managed snarare än återskapades.
Stack-resurser ökade från 196 till 225 i int-root, vilket inkluderar både nya filer och
återadopterade detached resources.

**Observation 4: Pre-existerande CI-brister identifierades under körningen.** Under T10
upptäcktes två orelaterade CI-problem: en typo `decomissioned` i workflow-genereringen
från acceleratorn (åtgärdat i v1.1.2) och avsaknaden av networking-hub what-if-jobb
eftersom acceleratorn ursprungligen bootstrappades med `network_type=none` (åtgärdat i
v1.1.3).

**En-meningskommentar:** Lib-bumpen `2025.09.2 → 2026.04.0` absorberades av plattformen
